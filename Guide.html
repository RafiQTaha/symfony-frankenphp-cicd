$
Installation + systemd
▼
BASH
gh api repos/TON_USER/symfony-frankenphp-cicd/actions/runners/registration-token \
  --method POST --jq '.token'
mkdir -p ~/cicd/actions-runner && cd ~/cicd/actions-runner
curl -o runner.tar.gz -L https://github.com/actions/runner/releases/download/v2.322.0/actions-runner-linux-x64-2.322.0.tar.gz
tar xzf runner.tar.gz
./config.sh --url https://github.com/TON_USER/symfony-frankenphp-cicd --token TON_TOKEN
sudo ./svc.sh install && sudo ./svc.sh start && sudo ./svc.sh status
systemd = démarrage automatique avec WSL2. Plus besoin de ./run.sh manuellement.

Pas de tests unitaires — que mettre à la place ?
▼

Ne jamais supprimer le job de vérification — remplace PHPUnit par une autre vérification :

YAML — alternatives à PHPUnit
# Minimum absolu — vérification syntaxe PHP
- name: Syntaxe PHP
  run: cd symfony && find src/ -name "*.php" -exec php -l {} \;

# PHPStan — détecte les bugs sans écrire de tests
- name: PHPStan
  run: cd symfony && vendor/bin/phpstan analyse src/ --level=5

# PHP CS Fixer — style de code
- name: Code style
  run: cd symfony && vendor/bin/php-cs-fixer fix --dry-run --diff

# Audit sécurité des dépendances
- name: Security audit
  run: cd symfony && composer audit
Recommandation : Même sans PHPUnit, ajoute au minimum composer audit + vérification syntaxe. Déployer sans aucune vérification = risque élevé.
